### Code review observations

1. **`EXEMPT_PREFIXES` is a dead constant.** The frozenset exists in `session.py` but is never evaluated at runtime — exemptions work because public routes simply don't declare `Depends(require_user)`. The engineer and the ticket both document this intentional design. Acceptable.

2. **`_try_refresh` error handling is correctly layered.** `_try_refresh` in `auth/oauth.py` catches only `requests.HTTPError` (Yahoo 401 → returns `None`). The middleware wraps the call in `except requests.RequestException` to also catch network-level failures (timeout, DNS). Both failure paths result in `new_tokens = None` → session deleted → `RequiresLogin` raised. Verified by `test_refresh_failure_redirects_and_deletes_session` and `test_network_error_on_refresh_redirects`.

3. **DB commit order is safe.** On refresh failure, `DELETE` + `commit()` happens before `raise RequiresLogin()` — no risk of an uncommitted delete if the exception propagates. On refresh success, `UPDATE` + `commit()` before returning the `CurrentUser`. Correct.

4. **`CurrentUser` dataclass fields match the ticket spec.** `session_id: str`, `access_token: str`, `expires_at: float` — all present.

5. **`web/main.py` registers the `RequiresLogin` exception handler correctly.** Returns `RedirectResponse("/auth/login", status_code=302)`. Import of `RequiresLogin` is present.
