### Manual verification

Manual browser verification is not possible in this environment (CLI-only). The engineer also flagged this limitation. The following observations are based on static template and route analysis:

**Template structure inspection (`head_to_head.html`):**
- Extends `base.html` correctly; no nav changes to base.
- Back link `← Back to Leaderboard` with `href="/overview"` is at top of content block.
- HTMX form attributes: `hx-get="/overview/head-to-head/table"`, `hx-target="#head-to-head-table"`, `hx-trigger="change"` — correct for partial update on any dropdown change.
- Four `<select>` elements: `name="team_a"`, `name="team_b"`, `name="from_week"`, `name="to_week"` — correct query param names matching the fragment route signature.
- Initial comparison rendered via `{% include "overview/_head_to_head_table.html" %}` inside a `<div id="head-to-head-table">` — HTMX will replace this same div on change.
- `not_enough_data` branch renders a message with no selectors; no crash path.

**Fragment template inspection (`_head_to_head_table.html`):**
- No `<html>`, `<head>`, or `<body>` tags — correct fragment.
- Winner highlighting logic matches spec exactly: green on winner side, gray on both sides for Tie, nothing on loser side.
- `Winner` column header and per-row `{{ row.winner }}` display correct.
- Tally line format: `{team_a}: {N} wins · {team_b}: {N} wins · {N} ties` — rendered via `&middot;` HTML entity.
- `{% if sim is not none and not sim.empty %}` guard prevents rendering when data is unavailable.

**Route logic inspection (`overview.py:119–216`):**
- `selected_league_name` passed in every shell context (AC: nav shows selected league) — confirmed on lines 131–133, 159.
- Fragment route correctly omits `selected_league_name` (fragment doesn't need it).
- `from_week > to_week` swap at line 201–202.

**Leaderboard page inspection (`index.html`):**
- "Compare two teams →" link is inside `{% if weeks %}` block — correct; only shown when data exists.

**`team_a == team_b` edge case (not tested, noted as acceptable per ticket):**
- `simulate()` on identical teams would produce averages where `a_val == b_val` for all categories → all "Tie". `tally()` returns `{team: 0, team: 0, "Tie": N}`. Template renders correctly. No crash path.

**Team names with spaces:**
- The `<select>` option `value="{{ t }}"` submits the exact team name string from the DataFrame. URL encoding of spaces is handled by the browser/HTMX. FastAPI's query param parsing decodes `%20` back to space before passing to the route handler. The round-trip is safe, but this was not exercised with a space-containing fixture. Noted below.

### Issues found

No FAILs against acceptance criteria. Two minor observations noted for the record:

**Observation 1 (not a blocker): Team names with spaces not covered by tests**
- The "Notes for QA" in the ticket specifically asks to test with at least one team name containing a space. All fixture team names are single words ("Alpha", "Beta", "Gamma"). The logic is correct by inspection (HTML `value` attribute + browser URL encoding + FastAPI query param decoding), but there is no automated exercise of this path.
- Severity: Minor — acceptable for v1; recommend adding a test case in a follow-up.

**Observation 2 (acknowledged): No end-to-end browser test**
- HTMX partial-update behaviour (table refreshes without full page reload) is structurally correct from template inspection but was not verified in a live browser. Engineer acknowledged this limitation. Structural correctness is high confidence.
- Severity: Minor — acceptable for v1.

**Observation 3 (by design): Demo mode not wired for head-to-head**
- `/overview/head-to-head` and `/overview/head-to-head/table` call `get_matchups` live; demo mode is not supported. Users accessing head-to-head without a session will be redirected to `/auth/login` as expected. Engineer explicitly deferred demo mode to a follow-up ticket.
- Severity: Out of scope for this ticket.
