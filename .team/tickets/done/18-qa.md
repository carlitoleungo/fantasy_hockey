### Issues found

**Bug 1: Demo route stat chips include player metadata columns**

- **Expected:** The stat chip checkboxes on `GET /demo/waiver` show only stat categories (Goals, Assists, Points, etc.) — the same stat names that would appear in the matchups scoring context.
- **Actual:** The stat chips include 5 player metadata columns as checkboxes: `player_key`, `player_name`, `team_abbr`, `display_position`, `status`. These appear before the real stat names. They have no abbreviation in `_STAT_FALLBACK_ABBREV`, so they render with their raw column names as labels.
- **Root cause:** `stat_columns()` (in `analysis/team_scores.py`) excludes a hardcoded set of matchups-DataFrame metadata columns: `{"team_key", "team_name", "week", "games_played"}`. `data.demo.load_season_pool()` returns a player-pool DataFrame whose metadata columns are `player_key`, `player_name`, `team_abbr`, `display_position`, `status` — none of which are in the exclusion set, so all 5 pass through as "stat columns."
- **Steps to reproduce:** `curl http://localhost:8765/demo/waiver | grep 'type="checkbox"'` — first 5 results are player_key, player_name, team_abbr, display_position, status.
- **Fix:** The demo route should derive `stat_cols` from `data.demo.get_matchups()` (a matchups-style DataFrame where `stat_columns()` works correctly), not from `load_season_pool()`. Replace the demo route's `df = demo_module.load_season_pool()` / `stat_cols = stat_columns(df)` with `stat_cols = stat_columns(demo_module.get_matchups())`.
- **Severity:** Major — the demo is the first thing unauthenticated users see; the broken filter panel undermines the product's first impression and could cause the POST handler (ticket 019) to receive invalid stat column names when these checkboxes are selected.

**Test coverage gap (not a separate bug, but related to Bug 1):**

`test_demo_waiver_shell_returns_200` (TC4) passes because it patches `data.demo.load_season_pool` with `_make_matchups_df()` — a matchups-style DataFrame whose columns (`team_key`, `team_name`, `week`, `games_played`, `Goals`, `Assists`, `Shots on Goal`) are exactly what `stat_columns()` was designed for. The test therefore never exercises the real `load_season_pool()` column structure. When the fix is applied (switching to `get_matchups()`), the test mock should be updated to patch `data.demo.get_matchups` instead.
