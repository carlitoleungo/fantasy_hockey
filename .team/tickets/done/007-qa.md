### Manual verification

This ticket has no UI component; the route returns a 302 redirect with no rendered HTML. Manual browser walk-through is not applicable. Verification was done via the test suite with an in-memory DB that allows direct inspection of `user_sessions` state after each call.

Key observations from reading the implementation (`web/routes/auth.py:94–103`):

1. `session_id` is read from the Cookie header using FastAPI's `Cookie(default=None)` pattern — consistent with the session middleware pattern.
2. `DELETE FROM user_sessions WHERE session_id = ?` is only executed when `session_id is not None`, preventing a SQL call on unauthenticated logout (correct).
3. `response.delete_cookie("session_id", secure=secure)` is called unconditionally — Set-Cookie header is always emitted regardless of whether a session existed.
4. `HTTPS_ONLY` env var is checked the same way as in `callback`, consistent with the existing pattern.
