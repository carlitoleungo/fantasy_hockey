### Observations (not AC failures — logged in `docs/improvements.md`)

**All-zero leaderboard on week 23 (championship week):** The overview defaults to the latest available week. During the championship period, many players haven't played, so the API returns `'-'` for their stats (coerced to `0` by `data/matchups`). The table renders correctly but shows all zeros with no useful ranking. Not a regression — existing data-layer behaviour. Logged for a future ticket (smart default week selection, or inline empty-state per week).

**Logout UX and silent re-auth:** After logout, the user lands on `/` with no "logged out" confirmation — the page looks identical to a logged-in user with no league selected. The session DB record is deleted on logout but the browser cookie persists, so clicking "Overview" silently triggers a full Yahoo OAuth round-trip that returns the user to `/` — appears as if nothing happened. Not a ticket 015 regression (auth flow predates this ticket), but the new nav link made it newly visible. Logged for a future ticket (clear cookie on logout, or add a post-logout landing page).

**`.team/LEARNINGS.md` does not exist.** The QA instructions require reading it before starting. Process gap, not a ticket defect.
